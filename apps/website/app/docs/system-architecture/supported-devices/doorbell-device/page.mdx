# Doorbell Device

<Callout type="info">
	**Device Category**: `doorbell`
</Callout>

The Doorbell device represents a smart doorbell system that provides video, audio, and additional monitoring features for enhanced home security.

---

## Required Channels

These channels are mandatory for the functionality of the doorbell:

| **Channel** | **Description** | **Multiple** | **Details** |
|-------------|-----------------|--------------|-------------|
| `doorbell` | Represents smart doorbells, providing features such as button press detection and optional camera integration. | No | [See details](/docs/system-architecture/supported-channels/doorbell-channel) |
| `device_information` | Provides metadata about the device, including manufacturer details, model information, and operational statuses. | No | [See details](/docs/system-architecture/supported-channels/device-information-channel) |

---

## Optional Channels

These channels are optional and provide additional features:

| **Channel** | **Description** | **Multiple** | **Details** |
|-------------|-----------------|--------------|-------------|
| `battery` | Reports the status of a device's battery, including level, charging state, and operational health. | No | [See details](/docs/system-architecture/supported-channels/battery-channel) |
| `camera` | Provides video and image data streams from surveillance or monitoring cameras. This channel supports configuration and control of camera devices, such as surveillance cameras or babysitting monitors. | No | [See details](/docs/system-architecture/supported-channels/camera-channel) |
| `contact` | Tracks the open or closed state of a contact point, such as a door, window, or cabinet. | No | [See details](/docs/system-architecture/supported-channels/contact-channel) |
| `light` | Controls lighting devices, supporting brightness, color temperature, and color adjustments. | No | [See details](/docs/system-architecture/supported-channels/light-channel) |
| `lock` | Manages the locked/unlocked state of physical locks for doors, safes, or other secure areas. | No | [See details](/docs/system-architecture/supported-channels/lock-channel) |
| `microphone` | Represents audio input devices, reporting activity or capturing sound. | No | [See details](/docs/system-architecture/supported-channels/microphone-channel) |
| `motion` | Detects motion in an area to trigger alerts or automate responses. | No | [See details](/docs/system-architecture/supported-channels/motion-channel) |
| `speaker` | Controls speaker devices, including volume, mute state, and audio output modes. | No | [See details](/docs/system-architecture/supported-channels/speaker-channel) |

---

## Use Case Scenarios

1. **Visitor Detection**:
    - Receive notifications when someone presses the doorbell or motion is detected.

2. **Video Communication**:
    - View live video feed and communicate with visitors using camera, microphone, and speaker channels.

3. **Access Control**:
    - Integrate with lock channel to remotely unlock doors for trusted visitors.

---

## Developer Notes

1. The doorbell channel is required for button press detection.
2. Camera integration is optional but common for video doorbells.
3. Lock channel enables remote access control functionality.
