# Media Device

<Callout type="info">
	**Device Category**: `media`
</Callout>

The Media device represents a multimedia system capable of playing, recording, or managing audio and video content.

---

## Required Channels

These channels are mandatory for the functionality of the media:

| **Channel** | **Description** | **Multiple** | **Details** |
|-------------|-----------------|--------------|-------------|
| `media_playback` | Provides playback controls for multimedia devices, excluding volume control. | No | [See details](/docs/system-architecture/supported-channels/media-playback-channel) |
| `device_information` | Provides metadata about the device, including manufacturer details, model information, and operational statuses. | No | [See details](/docs/system-architecture/supported-channels/device-information-channel) |

---

## Optional Channels

These channels are optional and provide additional features:

| **Channel** | **Description** | **Multiple** | **Details** |
|-------------|-----------------|--------------|-------------|
| `switcher` | Optional power control for media devices (hard power / standby) when supported by the integration. | No | [See details](/docs/system-architecture/supported-channels/switcher-channel) |
| `media_input` | Manages the input sources of multimedia devices like TVs, projectors, or audio receivers. | No | [See details](/docs/system-architecture/supported-channels/media-input-channel) |
| `microphone` | Represents audio input devices, reporting activity or capturing sound. | No | [See details](/docs/system-architecture/supported-channels/microphone-channel) |
| `speaker` | Controls speaker devices, including volume, mute state, and audio output modes. | No | [See details](/docs/system-architecture/supported-channels/speaker-channel) |

---

## Use Case Scenarios

1. **Playback Control**:
    - Control media playback with play, pause, skip, and seek functions. View current track information and progress.

2. **Input Selection**:
    - Switch between different media sources using the media_input channel.

3. **Audio Control**:
    - Adjust volume and mute state using the speaker channel.

---

## Developer Notes

1. The media_playback channel is required for playback control.
2. Switcher channel provides optional power control.
3. Speaker channel is recommended for volume control.
