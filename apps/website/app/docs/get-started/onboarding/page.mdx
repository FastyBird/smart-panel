# Onboarding

After installation, Smart Panel needs an **admin account** before you can start managing devices. The
onboarding process creates your first user and walks you through initial setup.

---

## Admin Interface Onboarding (Recommended)

The easiest way to onboard — open the admin UI in your browser and follow the guided wizard.

<Steps>
	### Open the Admin Interface

	Navigate to your Smart Panel in a browser:

	```
	http://smart-panel.local:3000
	```

	Or use the device's IP address: `http://<device-ip>:3000`

	### Create Your Admin Account

	On first visit, the onboarding wizard appears automatically. Enter:

	- **Username** — your admin login name
	- **Password** — a secure password (minimum 8 characters)
	- **Email** — for account recovery (optional)

	### Configure Your Home

	The wizard guides you through basic setup:

	- **Integrations** — connect your smart home platforms (Home Assistant, Shelly, Zigbee2MQTT, etc.)
	- **Device discovery** — automatically find devices on your network
	- **Spaces** — organize devices into rooms and areas

	<Callout type="info">
		You can skip any step and configure it later from the admin interface.
	</Callout>

	### Done

	After onboarding, you're taken to the admin dashboard where you can manage devices, create
	scenes, customize displays, and more.
</Steps>

---

## CLI Onboarding (Alternative)

If you prefer the command line, or need to create the admin account before accessing the web interface:

<Tabs items={['Install Script / npm', 'Tarball / Image']}>
	<Tabs.Tab>
		```bash copy
		sudo smart-panel-service onboarding yourusername yourpassword
		```
	</Tabs.Tab>
	<Tabs.Tab>
		```bash copy
		cd /opt/smart-panel/current   # or /opt/smart-panel for tarball installs
		node dist/main.js auth:onboarding yourusername yourpassword
		```
	</Tabs.Tab>
</Tabs>

<Callout type="warning">
	Replace `yourusername` and `yourpassword` with your actual credentials. Use a strong password.
</Callout>

After creating the account, open the admin interface to continue setup.

---

## Display Pairing

The panel display app connects to the backend automatically:

- **Pre-built images (AIO)** — the display is already paired with the local backend
- **Separate display devices** — on first launch, the display discovers the server via mDNS or prompts
  you to enter the server URL manually

No manual pairing steps are required — the display fetches its configuration from the backend
and starts showing the dashboard.
