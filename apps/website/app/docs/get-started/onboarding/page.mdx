# Onboarding

Before you can start managing devices or customizing your dashboard, your Smart Panel needs to be **linked to a user account**.
This process ensures secure access and allows your panel to communicate with backend services.

If you installed via the **one-liner script** or **npm**, the install process may have already prompted you to create
an owner account. If not, or if you used the **manual tarball method**, follow the steps below.

In a future update, a web-based onboarding interface will make this even easier — stay tuned!

---

## Registering the Owner

The command to register an owner depends on how you installed Smart Panel:

**If you installed via npm (globally):**

```bash copy
sudo smart-panel auth:onboarding yourusername secretpassword
```

**If you installed via tarball (manual):**

```bash copy
cd /opt/smart-panel
npm run cli auth:onboarding yourusername secretpassword
```

<Callout type="info">
	Use a secure username and password combination — these will be used to log in to the admin interface.
</Callout>

<Callout type="warning">
	Replace `yourusername` and `secretpassword` with your actual credentials. Do not use default or weak passwords.
</Callout>

If everything goes well, you'll see:

```bash
✅ Successfully created owner account: yourusername
```

---

## Display Onboarding

Once the display service is started (as shown in the previous [installation](/docs/get-started/installation) step), the **Display UI**
will automatically boot into its default onboarding state.

There is **no manual onboarding required** for the display at this stage. It will:

- Connect to the backend
- Fetch configuration
- Show a default splash screen or pairing status

<Callout type="info">
	Future versions will include a guided UI onboarding wizard.
</Callout>
