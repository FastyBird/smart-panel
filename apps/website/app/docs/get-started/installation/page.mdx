# Installation

Now that your device is set up, it's time to install the Smart Panel software. Depending on your setup,
you can choose one of three deployment scenarios.

All installation methods use **pre-built distribution packages** published automatically by our CI pipeline
to [GitHub Releases](https://github.com/FastyBird/smart-panel/releases).

---

## Deployment Scenarios

<Cards>
	<Cards.Card title="Server Only" href="/docs/get-started/installation/server-only">
		Install just the backend and admin interface on a server or headless device. Display panels connect remotely.
	</Cards.Card>
	<Cards.Card title="Display Only" href="/docs/get-started/installation/display-only">
		Install the panel app on a display device that connects to an existing Smart Panel server on your network.
	</Cards.Card>
	<Cards.Card title="All-in-One" href="/docs/get-started/installation/all-in-one">
		Run the server and display on the same device — ideal for a standalone Raspberry Pi smart panel.
	</Cards.Card>
	<Cards.Card title="Docker" href="/docs/get-started/installation/docker">
		Deploy the server as a Docker container — quick setup with no system-level dependencies.
	</Cards.Card>
</Cards>

## Which Scenario Is Right for You?

| Scenario | Best For | Example Hardware |
|----------|----------|-----------------|
| **Server Only** | Centralized setup with multiple displays | Raspberry Pi 4, mini PC, NAS, any Linux server |
| **Display Only** | Adding a screen to an existing server | Raspberry Pi Zero 2W + touchscreen, Android tablet |
| **All-in-One** | Single standalone smart panel | Raspberry Pi 3B+/4 with DSI touchscreen |
| **Docker** | Containerized server deployment | Any Linux host with Docker installed |

<Callout type="info">
	Not sure which to pick? If you have **one Raspberry Pi with a display**, go with
	[All-in-One](/docs/get-started/installation/all-in-one). If you plan to have **multiple displays
	around your home**, start with [Server Only](/docs/get-started/installation/server-only) on a central
	device, then add displays using [Display Only](/docs/get-started/installation/display-only).
</Callout>

## System Requirements

### Server (Backend + Admin)

- **OS**: Linux with systemd (Raspberry Pi OS, Debian, Ubuntu, Fedora, etc.)
- **Node.js**: Version 24 or higher
- **Architecture**: ARM (32-bit), ARM64 (64-bit), or x64
- **RAM**: 512 MB minimum, 1 GB recommended
- **Storage**: 300 MB free space minimum

### Display (Panel App)

- **Raspberry Pi**: flutter-pi runtime (no desktop environment needed)
- **Linux x64**: eLinux DRM-GBM mode (no desktop required) or Linux desktop with GTK3
- **Android**: Android 5.0+ (sideloaded via ADB or APK)
