# reTerminal Hardware Plugin

The **reTerminal plugin** provides native hardware integration for SeeedStudio reTerminal devices when Smart Panel runs directly on the hardware.

## What It Does

This plugin gives the Devices module direct access to the onboard sensors, LEDs, buttons, and other peripherals built into SeeedStudio reTerminal hardware. Once active, you can:

- Monitor onboard sensor data (light level, accelerometer orientation, CPU temperature)
- Control LEDs and the buzzer
- React to physical button presses (including long-press and double-press)
- Display hardware status on the dashboard

## Supported Hardware

### reTerminal CM4

The original **reTerminal** with a 5" capacitive touchscreen and Raspberry Pi Compute Module 4:

- **4 programmable buttons** (F1, F2, F3, O) with press, long-press, and double-press detection
- **USR LED** (green, controllable)
- **STA LED** (red/green, dual-color)
- **Buzzer** output
- **Light sensor** (LTR-303ALS)
- **Accelerometer** (LIS3DHTR) with orientation detection
- **CPU temperature** monitoring

### reTerminal DM

The **reTerminal DM** with a 10.1" industrial display:

- **Status LED**
- **CPU temperature** monitoring

<Callout type="info">
	The reTerminal DM does not include buttons, a buzzer, or additional sensors.
</Callout>

## How It Works

- The plugin **auto-detects** the reTerminal variant by reading `/proc/device-tree/model` at startup.
- It automatically creates a device with the appropriate channels and properties for the detected variant.
- No manual device creation is needed — the plugin handles everything based on the detected hardware.
- On non-reTerminal hardware, the plugin enters **standby mode** — no errors, just inactive.

## Communication Interfaces

The plugin communicates with the hardware through standard Linux interfaces:

| Interface | Used For |
|-----------|---------|
| `/sys/class/leds/` | LED and buzzer control |
| `/dev/input/` | Button event monitoring (evdev) |
| `/sys/bus/iio/devices/` | Light sensor and accelerometer |
| `/sys/class/thermal/` | CPU temperature |

## Configuration

You can enable or disable the reTerminal plugin in the Admin App:

- Navigate to **Extensions → reTerminal Devices** in the admin interface.
- Toggle the plugin on or off.
- The sensor polling interval is configurable.

<Callout type="info">
	You must restart the backend for configuration changes to take effect.
</Callout>

## Auto-Rotate (Experimental)

The reTerminal CM4 includes a built-in accelerometer (LIS3DHTR) that can detect screen orientation. Smart Panel ships with an optional **auto-rotate daemon** that monitors the accelerometer and automatically rotates the display when the device orientation changes.

<Callout type="warning">
	Auto-rotate requires restarting the flutter-pi display process on each rotation, which causes a brief screen reload (~1-2 seconds). This feature is **disabled by default** and must be explicitly enabled.
</Callout>

### How It Works

1. The daemon polls the accelerometer at `/sys/devices/platform/lis3lv02d/position`
2. When a stable orientation change is detected (after debounce), it updates `FB_DISPLAY_ROTATION` in `/etc/smart-panel/display`
3. The `smart-panel-display` service is restarted with the new rotation value

### Enabling Auto-Rotate

Edit the display configuration file:

```bash
sudo nano /etc/smart-panel/display
```

Set `FB_AUTO_ROTATE=1`:

```bash
FB_AUTO_ROTATE=1
```

Then enable and start the service:

```bash
sudo systemctl enable smart-panel-auto-rotate.service
sudo systemctl start smart-panel-auto-rotate.service
```

### Configuration Options

All tunables are set in `/etc/smart-panel/display`:

| Variable | Default | Description |
|----------|---------|-------------|
| `FB_AUTO_ROTATE` | `0` | Enable (`1`) or disable (`0`) auto-rotation |
| `FB_AUTO_ROTATE_POLL_INTERVAL` | `1` | Accelerometer polling interval in seconds |
| `FB_AUTO_ROTATE_DEBOUNCE` | `3` | Consecutive identical readings required before rotating |
| `FB_AUTO_ROTATE_THRESHOLD` | `400` | Minimum tilt in milligravity to detect a valid orientation |

### Orientation Mapping

| Physical Orientation | Rotation |
|---------------------|----------|
| Landscape (default, USB ports down) | 0° |
| Portrait (rotated left) | 90° |
| Landscape inverted (USB ports up) | 180° |
| Portrait inverted (rotated right) | 270° |
| Flat (face up / face down) | Unchanged |

<Callout type="info">
	Auto-rotate is only available on the **reTerminal CM4**. The reTerminal DM does not have an accelerometer.
</Callout>

### Troubleshooting

Check the daemon logs:

```bash
journalctl -u smart-panel-auto-rotate -f
```

If the service fails to start, verify:
- The accelerometer is accessible: `cat /sys/devices/platform/lis3lv02d/position`
- `FB_AUTO_ROTATE=1` is set in `/etc/smart-panel/display`
- The display service is running: `systemctl status smart-panel-display`
